# 执行流水线与输出规范

## 概述

定义热门题材挖掘的完整执行步骤、各步骤间的数据流转关系，以及最终报告的输出格式规范。

> **数据获取统一通过 [[theme_miner_data_bridge]]**：所有 A 股数据来自 a-stock-data 各层的内嵌 Python 函数（约定 D：模型读 reference → `python3 -c` 执行 → 返回 Python 值），涨停/跌停/市场情绪由 data_bridge 的补充端点提供。**所有东财请求走 `em_get()`**（防封，见 `a_stock_data_common.md`）。

---

## 执行流水线

```
Step 1: 市场情绪采集
   │
   ├── 涨停数/跌停数 ← data_bridge 涨停池/跌停池补充端点
   ├── 上涨/下跌家数 ← data_bridge 由 a-stock-data 行业板块 f104/f105 汇总
   ├── 计算市场情绪评级
   └── 输出: 上涨家数、下跌家数、涨停数、情绪评级
   │
Step 2: 题材数据采集
   │
   ├── 概念板块/行业板块行情 ← a-stock-data a_stock_signals（东财 push2 m:90+t:2）
   ├── 涨停股列表及 hybk 行业归属 ← data_bridge 涨停池
   ├── 将涨停股按行业/题材标签匹配到各板块
   └── 输出: 每个板块的涨幅、成交额、换手率、涨跌家数、涨停家数
   │
Step 3: 题材评分与 Top3 筛选
   │
   ├── 对每个板块执行三维评分（参见 [[theme_miner_theme_scoring]]）
   ├── 判定生命周期阶段和题材级别
   ├── 按综合得分降序排序，取 Top3
   └── 输出: Top3 题材（含得分、驱动逻辑、操作策略）
   │
Step 4: 成分股数据采集
   │
   ├── Top3 题材成分股 ← a-stock-data a_stock_signals 概念板块成分/行业排名
   ├── 对每个题材取涨幅前 10-15 只候选股
   ├── 候选股资金流向 ← a-stock-data a_stock_capital_flow / a_stock_signals
   ├── 候选股基本面 ← a-stock-data a_stock_fundamentals / 腾讯 tencent_quote
   └── 输出: 候选股完整数据集
   │
Step 5: 个股评分与 Top5 筛选
   │
   ├── 对候选股执行五维评分（参见 [[theme_miner_stock_scoring]]）
   ├── 全部候选股按综合得分降序排序，取 Top5
   ├── 标注龙头地位、技术形态、资金趋势
   └── 输出: Top5 个股（含五维得分及详细数据）
   │
Step 6: 价格预测与策略生成
   │
   ├── 对 Top5 个股计算中长期目标价（参见 [[theme_miner_price_prediction]]，⚠️ 启发式）
   ├── 生成三情景预测（乐观/中性/悲观）
   ├── 计算预测置信度
   ├── 生成操作策略（仓位、入场、止损、出场）
   ├── 生成风险提示
   └── 输出: 完整预测与策略
   │
Step 7: 报告组装
   │
   ├── 组装市场情绪模块
   ├── 组装 Top3 题材模块
   ├── 组装 Top5 个股模块（含预测与策略）
   ├── 组装风险提示模块
   ├── 添加免责声明（含价格模型启发式说明）
   └── 输出: 完整 Markdown 报告
```

## 市场情绪评级标准

| 评级 | 条件 |
|:---|:---|
| **高潮** | 涨停数 ≥ 80 且 上涨家数/下跌家数 ≥ 3 |
| **活跃** | 涨停数 ≥ 40 且 上涨家数 > 下跌家数 |
| **中性** | 上涨家数 与 下跌家数 相差不大（0.8-1.2 倍） |
| **低迷** | 下跌家数 > 上涨家数 且 涨停数 < 20 |
| **冰点** | 涨停数 < 10 且 下跌家数/上涨家数 ≥ 3 |

## 数据流转约束

### 请求频率控制

所有东财请求统一走 a-stock-data 的 `em_get()`（串行限流 `EM_MIN_INTERVAL=1.0s` + 抖动 + 会话复用），天然满足以下约束，无需额外 sleep：

| 操作 | 约束 |
|:---|:---|
| 概念板块/行业板块列表 | 单次请求，获取全部板块 |
| 涨停池/跌停池 | 各单次请求 |
| 板块成分股 | 每个 Top3 板块请求一次（共 3 次） |
| 个股资金流向 | 每只候选股请求一次（em_get 自动节流） |
| 个股基本面 | 每只候选股请求一次（em_get 自动节流） |

> 注：候选股数量大时（如每题材 15 只 × 3 = 45 只 × 2 项数据 = 90 次东财请求），按 em_get 的 1s 间隔约需 1.5 分钟，属正常。腾讯 `tencent_quote` 基本面/行情不走 em_get（腾讯不封 IP），可批量加速。

### 数据缺失处理

| 缺失数据 | 处理方式 |
|:---|:---|
| 板块行情数据获取失败 | 报告错误，终止流程 |
| 涨停股列表获取失败 | 所有板块涨停数按 0 处理，继续评分；情绪评级降级用涨跌比 |
| 个股资金流向获取失败 | 资金面维度得分设为基础分，资金趋势标记为「数据不足」 |
| 个股基本面获取失败 | 基本面维度得分设为基础分，PE/营收/净利标记为「--」 |
| 价格数据缺失 | 跳过该个股价格预测，标注「数据不足」 |

## 与其他层 / Skill 的协作（mx-skills 实际映射）

| 层 / Skill | 关系 | 协作方式 |
|:---|:---|:---|
| **a-stock-data**（`a_stock_*`） | **上游数据依赖（必需）** | 提供全部 A 股数据：概念/行业板块、成分股、资金流、基本面、强势股归因。单向依赖，本层引用其函数，a-stock-data 不感知本层 |
| **mx-skills #12 热点发现** | 同域、互为备选 | 简单「今天什么热」优先走 #12（付费 API，快）；本层用于整条 题材→个股→目标价→策略 链路，或 #12 配额耗尽时降级 |
| **mx-skills #10 股票诊断** | 下游扩展 | 对 Top5 个股可进一步调用做单票综合诊断 |
| **mx-skills #1 金融问答 / #3 资讯搜索** | 下游补充 | 对入选题材/个股可补充资讯、研报、催化事件核实 |
